# Monad Laws Proof Structure Design

You are an expert in Coq and type theory.

We consider an abstract type constructor:

M : Type -> Type

with operations:

return : forall {A : Type}, A -> M A
bind : forall {A B : Type}, M A -> (A -> M B) -> M B

We want to express in Coq that (M, return, bind) forms a monad,
and we want to provide a proof skeleton for the monad laws:

1. Left identity
2. Right identity
3. Associativity

Your tasks:

1. Design a Coq typeclass `Monad` that encapsulates `M`, `return`, and `bind`,
   including the monad laws as fields or as separate propositions.

2. Show how you would structure the proofs of each law:
   - what the statement of each theorem looks like,
   - what proof strategy / tactics you would use (e.g., `intros`, `rewrite`, `reflexivity`,
     or custom tactics),
   - where you would expect difficulties for more complex instances (e.g. state monad, I/O monad).

3. Provide at least one concrete example instance (e.g. option monad or list monad),
   and outline how the monad laws would be discharged for that instance.

4. Please write the Coq code in a reasonably realistic style, but you do NOT have
   to produce fully type-checked code. Focus on:
   - clear structure,
   - reasonable use of universes and implicit arguments,
   - modularity of definitions.

Answer in English.
