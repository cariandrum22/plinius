# Type-Level Secret Key Logging Prevention

You are an expert in dependent types and formal verification
using either Coq or F\*.

We have a logging API for an application that manipulates cryptographic keys:

- type SecretKey
- type PublicKey
- type LogMessage

We want to design a type system (in Coq or F\*) such that:

"It is impossible to construct a value of type LogMessage
that directly or indirectly contains any SecretKey."

Tasks:

1. Propose a type-level design that enforces this property:
   - e.g. a phantom type parameter, refinement types, or a GADT-style encoding,
   - separation between "tainted" and "untainted" data.

2. Provide type signatures for:
   - functions that create `LogMessage` from safe inputs,
   - functions that manipulate `SecretKey` but cannot produce `LogMessage`.

3. Show how you would encode and prove a key lemma such as:

   "For all terms `t` well-typed under this system, any `LogMessage` reachable
   from `t` is free of `SecretKey` data."

4. Describe the proof strategy:
   - induction on typing derivations or program structure,
   - use of invariants,
   - how to maintain this guarantee under program extension (new modules/functions).

You may write pseudo-code in Coq or F\* style. Focus on the structure of the argument.
Answer in English.
