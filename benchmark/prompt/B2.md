# F\* Dijkstra Monad for Non-Interference

You are an expert in F\* and program verification.

We consider a function:

val f : input:state -> output \* state

where `state` contains both:

- a "secret" field (e.g. secret_key : bytes)
- a "public" field (e.g. public_state : int)

We want to express and prove in F\* that `f` satisfies a non-interference property:

Changes in the secret field should not influence the observable public outputs.

Tasks:

1. Describe how you would model this function using a Dijkstra monad in F\*,
   with a precondition and postcondition capturing non-interference.

2. Define (in pseudo-F\* code) the relevant types and predicates:
   - a state type with secret and public components,
   - a relation "public_eq" that equates two states on their public parts only,
   - a predicate that formalizes non-interference for `f`.

3. Sketch the proof structure:
   - what lemmas you would need,
   - how you would structure the main theorem,
   - how you would use F\*'s SMT automation and manual lemmas,
   - which parts are likely to require user-guided reasoning.

4. Explain how this approach would scale to a module with multiple functions
   interacting on the same state (e.g. init, update, rotate_key, log_event).

Write your answer in English and include F\*-like pseudo-code where helpful.
